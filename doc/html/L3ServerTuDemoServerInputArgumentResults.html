Setting InputArgumentResults

If an OPC UA Method is called by a client, the input arguments need to be validated by the server. If an argument is invalid, this information can be returned to the client by setting the InputArgumentResults. The SDK implicitly checks the DataType and the ValueRank of each argument and sets the InputArgumentResults automatically. Furthermore the content of each input argument can be checked by the application.

When implementing the methods by implementing the CallMethodEventHandler the InputArguments can be set directly. The index in the list InputArgumentResults must match the index of the invalid argument. Set StatusCode BadOutOfRange to indicate that an input argument is outside the acceptable range.

private StatusCode DoOptionSetByteTest(
RequestContext context,
MethodHandle methodHandle,
IList<Variant> inputArguments,
List<StatusCode> inputArgumentResults,
List<Variant> outputArguments)
{
byte inOptionSetByte = inputArguments[0].ToByte();
if (inOptionSetByte >= ((byte) Model.OptionSetByte.CONTENT) * 2)
{
inputArgumentResults[0] = StatusCodes.BadOutOfRange;
return StatusCodes.BadInvalidArgument;
}

When using generated method stubs to implement an OPC UA Method, you can throw an MethodCallArgumentException. The argumentName must match the Name one of the input arguments.

public StatusCode Fill(RequestContext context, UnifiedAutomation.Demo.Model.BoilerModel model, double SetPoint)
{
lock (model)
{
if (SetPoint > model.FillLevelSensor.FillLevel.EURange.High)
{
throw new MethodCallArgumentException(nameof(SetPoint), "Requested SetPoint too high.");
}
if (SetPoint < model.FillLevelSensor.FillLevel.EURange.Low)
{
throw new MethodCallArgumentException(nameof(SetPoint), "Requested SetPoint too low.");
}
model.FillLevelSetPoint = SetPoint;
}
return StatusCodes.Good;
}

This feature can be tested with UaExpert by calling the Method Demo > 009_BoilerDemo > Boiler1 > Fill with a SetPoint > 100.
