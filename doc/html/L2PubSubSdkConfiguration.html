PubSub configuration of servers

Configuration through a client application

The PubSub configuration of a server is usually prepared or modified via the standard PubSubConfigurationFileType interface. This allows the online configuration with a tool like the UaExpert. To avoid losing the configuration after a restart, it is persisted when changes are made and restored at startup. The path for the file must be provided here by the PubSubSettings. Alternatively, you can also implement the LoadPubSubConfiguration and PersistPubSubConfiguration methods yourself if, for example, the configuration is not to be saved in the file system but in a database.

Note
The referenced nodes must already be created for the persisted configuration to be restored correctly. Usually the node and namespace creation is done in the OnRootNodeManagerStarted method, which is called before the PubSub configuration restoration.

Static configuration

In some cases, the flexibility of an online configuration is not desired at all, but the configuration should be delivered permanently with the server. The online configuration mechanism can be turned off by setting the CloseAndUpdate method of the PubSubConfiguration object to non-executable. This can be achieved with the InternalClient

InternalClient.WriteAttribute(DefaultRequestContext, MethodIds.PublishSubscribe_PubSubConfiguration_CloseAndUpdate, Attributes.Executable, false);

The fixed configuration can be provided by overriding the LoadPubSubConfiguration method. The PersistPubSubConfiguration method should be overridden to do nothing.
